## Goal
Verify the single-todo delete flow for one open todo.

## Scope
- Deliverable: manual QA checklist only.
- Delete flow covered: delete one open todo.
- Storage location covered: current working directory `.pi/todos` only.
- Clear all behavior expectation for related reference: Clear all should delete all todos, including closed/done, but this checklist focuses on deleting one open todo.
- Failure behavior expectation for related reference: if delete fails, stop immediately and show an error.
- Auto-cleanup reference: completed-todo cleanup should use existing `created_at`, but this checklist does not validate auto-cleanup directly.
- UI expectation after deletion: return to the main Todos menu.

## Manual QA checklist
1. Create a new open todo in the current working directory `.pi/todos`.
2. Open `/todos`.
3. Choose `View all`.
4. Select the open todo.
5. Choose the delete action for that todo.
6. Confirm deletion.
7. Verify the todo file is removed from `.pi/todos`.
8. Verify the todo no longer appears in `View all`.
9. Verify the UI returns to the main Todos menu after deletion.
10. Verify the widget updates immediately and no longer shows the deleted todo.

## Edge case to include
- No todos: after deleting the only open todo, confirm `View all` shows no matching todos / empty state appropriately.

## Out of scope
- Deleting completed/done todos.
- Clear all flow.
- Global `~/.pi/agent/todos` storage.
- Assigned/running todos.
- Corrupt, locked, missing todo files.
- Many-todo performance scenarios.
- Automated tests or implementation changes.
